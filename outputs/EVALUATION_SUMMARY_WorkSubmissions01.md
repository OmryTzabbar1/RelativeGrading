## Grading Methodology

### Base Score Calculation
```
Base Score = (sum of weighted criteria student has) / (total possible weight) √ó 100
Max possible weight: 12.81 (sum of all criterion weights)
```

### Rarity Bonus System
Students receive +1 point for each criterion with ‚â§15% prevalence (‚â§5 students)

**Rationale:** Rewards students who implemented rare/distinguishing features beyond common requirements

### Final Score
```
Final Score = Base Score + Sum of Rarity Bonuses (capped at 100)
```

## Capabilities-Based Extraction

Following the updated EXTRACTION.md rules, this evaluation focused on **capabilities accomplished** rather than **technology choices**:

### ‚úÖ What We Extracted (Capabilities)
- Frontend implementation (not "React" or "Vue")
- Backend API (not "FastAPI" or "Express")
- Database integration (not "PostgreSQL" or "MongoDB")
- Testing infrastructure (unit, integration, coverage)
- Documentation artifacts (PRD, Architecture, API docs)
- Error handling and data validation
- Configuration management

### ‚ùå What We Did NOT Extract (Tech Choices)
- "Built with React"

- "Uses PostgreSQL"
- "Written in Python"
- Specific library/framework names

**Result:** Fair comparison where students using different tech stacks to achieve the same capabilities are evaluated equally.
