# Output File Formats

This document specifies the format for each output file generated by the evaluator.

## Output Directory

All files are saved to the `outputs/` directory in the project root.

```
outputs/
├── criteria_graph.json      # Complete criteria data
├── flagged_criteria.md      # Uncategorized items for review
├── grades.csv               # Student grades and ranks
└── evaluation_report.md     # Full analysis report
```

---

## 1. criteria_graph.json

Complete criteria data with all discovered criteria, students, weights, and categories.

### Schema

```json
{
  "metadata": {
    "generated_at": "2025-12-17T10:30:00Z",
    "total_students": 35,
    "total_criteria": 89,
    "categorized_count": 82,
    "uncategorized_count": 7,
    "master_folder": "/path/to/submissions"
  },
  "criteria": {
    "criterion_key": {
      "display_name": "Human Readable Name",
      "students": ["alice", "bob", "carol"],
      "count": 3,
      "weight": 0.086,
      "category": "Testing",
      "sources": {
        "alice": "README.md",
        "bob": "docs/TESTING.md",
        "carol": "README.md"
      }
    }
  },
  "categories": {
    "Documentation": {
      "criteria_count": 12,
      "total_weight": 8.5,
      "criteria": ["readme", "api_docs", "user_guide"]
    },
    "Testing": {
      "criteria_count": 8,
      "total_weight": 5.2,
      "criteria": ["unit_tests", "integration_tests", "coverage"]
    }
  },
  "students": {
    "alice": {
      "criteria_present": ["readme", "unit_tests", "ci_cd"],
      "criteria_count": 45,
      "raw_score": 38.5,
      "max_possible": 52.0,
      "percentage": 74.0,
      "grade": 92.5,
      "rank": 3
    }
  }
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `metadata.generated_at` | string | ISO timestamp of generation |
| `metadata.total_students` | number | Count of student folders processed |
| `metadata.total_criteria` | number | Unique criteria discovered |
| `criteria[key].display_name` | string | Human-readable criterion name |
| `criteria[key].students` | array | List of student names who have this |
| `criteria[key].count` | number | Number of students with this criterion |
| `criteria[key].weight` | number | Prevalence weight (count/total) |
| `criteria[key].category` | string | Assigned category or "Uncategorized" |
| `criteria[key].sources` | object | Which file each student's criterion came from |

## 2. flagged_criteria.md

Markdown file listing criteria that couldn't be categorized, for manual review.

### Format

```markdown
# Flagged Criteria for Manual Review

Generated: 2025-12-17

These criteria were extracted but could not be automatically categorized.
Please review and either:
1. Add them to an existing category in CATEGORIES.md
2. Create a new category
3. Mark as irrelevant (exclude from scoring)

---

## Uncategorized Criteria

| Criterion | Students | Count | Weight |
|-----------|----------|-------|--------|
| Blockchain integration | alice, bob | 2 | 0.057 |
| AI chatbot | carol, diana, eve | 3 | 0.086 |
| Custom logging framework | frank | 1 | 0.029 |
| GraphQL API | alice, bob, carol | 3 | 0.086 |

---

## Recommended Actions

### High Priority (3+ students)
These criteria appear in multiple projects and may deserve a category:
- AI chatbot (3 students)
- GraphQL API (3 students)

### Low Priority (1-2 students)
These are rare and may be project-specific:
- Blockchain integration (2 students)
- Custom logging framework (1 student)

---

## How to Categorize

1. Edit CATEGORIES.md
2. Add keywords under appropriate category
3. Re-run evaluation

Or manually edit criteria_graph.json to assign categories.
```

## 3. grades.csv

CSV file with student grades, suitable for import into grading systems.

### Format

```csv
Rank,Student,Grade,Percentage,Raw_Score,Max_Possible,Criteria_Count,Total_Criteria
1,alice,100.0,80.5,41.8,52.0,45,89
2,bob,96.2,77.4,40.2,52.0,43,89
3,carol,94.1,75.8,39.4,52.0,42,89
4,diana,91.8,73.9,38.4,52.0,41,89
5,eve,89.5,72.1,37.5,52.0,40,89
...
35,zack,62.3,50.2,26.1,52.0,28,89
```

### Column Descriptions

| Column | Type | Description |
|--------|------|-------------|
| Rank | int | Position (1 = highest grade) |
| Student | string | Student folder name |
| Grade | float | Final grade (0-100, best = 100) |
| Percentage | float | Raw percentage of criteria met |
| Raw_Score | float | Sum of weights for criteria present |
| Max_Possible | float | Sum of all criterion weights |
| Criteria_Count | int | Number of criteria this student has |
| Total_Criteria | int | Total unique criteria discovered |

### Notes

- Sorted by Grade descending
- Ties have same rank
- Grade is relative (best = 100)
- Percentage is absolute (could theoretically be 100 if student has everything)

## File Naming

If running multiple evaluations, append timestamps:

```
outputs/
├── criteria_graph_20251217_103000.json
├── flagged_criteria_20251217_103000.md
├── grades_20251217_103000.csv
└── evaluation_report_20251217_103000.md
```

Or ask user for custom suffix:
```
outputs/
├── criteria_graph_assignment1.json
├── grades_assignment1.csv
└── ...
```
